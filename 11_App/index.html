Week 11:

Application Programming

The homework for week 11 was to send data to a computer and visualize the results.

During class, I set up my esp32-cam using the following site:

https://randomnerdtutorials.com/program-upload-code-esp32-cam

Afterward, I copied the code from https://randomnerdtutorials.com/esp32-cam-opencv-js-color-detection-tracking and followed the instructions of the serial monitor. I was able to track my face using the camera and output a coordinate.


In order to show that the tracking worked, I tried to make it so if I move my head to a specific position, some light would turn on. This never worked out, however, and the light never turned on. I suspect this is because of the lag of the camera. Here is a video that shows how jittery the footage was.


To complete the assignment, I recorded the serial monitor, which displayed real-time the location of my head in the camera.


After reviewing the footage, I think it may be the code that is forcing the fps of the camera to decrease. To test this, I used another code that lets me use the camera on the esp32-cam.

I then followed the tutorial on randomnerd's page.

https://randomnerdtutorials.com/esp32-cam-video-streaming-face-recognition-arduino-ide

I was definitely correct. Using another code, the camera had way higher fps, as shown below.


I then tried to modify my tracking code for higher frames to no avail. If I had more time, I would try to add tracking to the new code.
